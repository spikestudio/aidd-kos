# Business Context: PyPI 公開・リリースパイプライン整備

Epic: #17
対象 Phase: Phase 1 (Core MVP)
作成日: 2026-06-04

---

## 業務目的

aidd-kos パッケージを PyPI（Python Package Index）に公開し、
オペレーターが `uvx aidd-kos install` という標準コマンドだけで導入できる状態を作る。
GitHub Release をリリース作業の起点とし、その作成が自動的に PyPI への公開へと繋がる仕組みを整備することで、
メンテナーの作業負担を減らしかつリリース漏れを防ぐ。

## ドメイン概念

| 用語 | 定義 | 補足 |
|------|------|------|
| PyPI（Python Package Index）| Python パッケージの公式リポジトリ。`pip install` / `uvx` でインストールする際の参照先 | glossary.md 参照 |
| GitHub Release | GitHub 上でタグと成果物を紐付けて公開する仕組み。`v*` パターンのタグ付きリリースがパブリッシュのトリガーとなる | glossary.md 参照 |
| パブリッシュ | aidd-kos のビルド成果物（wheel / sdist）を PyPI に登録し外部からインストール可能にすること | — |
| リリース | 新バージョンを GitHub Release として公開し PyPI に反映させる一連の作業 | バージョン番号更新・タグ作成・GitHub Release 作成・パブリッシュ完了の全工程 |
| バージョン番号 | aidd-kos パッケージの識別子（例: v0.1.0）。タグに付与し PyPI バージョン一覧にも反映される | — |
| リリース手順ドキュメント | メンテナーがリリース作業を実施するための手順書。`RELEASE.md` として管理される | — |

## 管理対象と状態遷移

### パッケージリリース

- 状態: `未公開（開発中）` → `リリース準備中` → `公開済み（PyPI に反映）` / `公開失敗`
- 遷移トリガー:
  - 未公開 → リリース準備中: メンテナーがバージョン番号を更新しタグを作成したとき
  - リリース準備中 → 公開済み: GitHub Release 公開をトリガーにパブリッシュが完了したとき
  - リリース準備中 → 公開失敗: パブリッシュ処理が失敗したとき
- 各状態の制約:
  - `未公開`: オペレーターは `uvx aidd-kos install` でインストールできない
  - `公開済み`: `uvx aidd-kos@latest` が最新バージョンを参照できる
  - `公開失敗`: メンテナーが手動でリカバリ作業を実施するまで新バージョンは利用不可

## ビジネスルール

| ID | ルール | 関連 AC |
|----|-------|--------|
| BR-DIST-01 | リリースは `v*` パターンのタグが付いた GitHub Release を作成することで開始する | AC-F[M]-01 |
| BR-DIST-02 | パブリッシュは GitHub Release 公開後に自動実行される（手動実行を標準としない）| AC-F[M]-01, AC-F[M]-02 |
| BR-DIST-03 | リリース手順書（RELEASE.md）には「バージョン番号の更新」「タグの作成」「GitHub Release の作成」の 3 手順が必ず含まれている | AC-F[L]-01 |
| BR-DIST-04 | `uvx aidd-kos@latest serve` は PyPI の最新公開バージョンを参照して起動する | AC-F[N]-02 |

## 例外・禁止事項

| ID | 条件 | システムの振る舞い |
|----|------|----------------|
| EX-DIST-01 | パブリッシュ処理が失敗した | GitHub Actions のワークフロー実行ステータスが "failure" となり、メンテナーが GitHub 上で検知できる |

## 業務イベント

| イベント名 | 発生条件 | 後続アクション |
|----------|---------|-------------|
| パッケージ公開完了 | GitHub Release 公開後にパブリッシュが正常完了したとき | `uvx aidd-kos@latest` で新バージョンがインストール可能になる |
| パブリッシュ失敗検知 | パブリッシュ処理が失敗したとき | メンテナーが GitHub でワークフローの失敗ステータスを確認し、手動でリカバリ作業を行う |
