# Business Context: Embedded 起動 & ストレージ移動

Epic: #3
対象 Phase: Phase 1 (Core MVP)
作成日: 2026-06-04

---

## 業務目的

MCP Server（aidd-kos）は起動時に LightRAG（ナレッジエンジン）を自動起動し、停止時に自動停止する。
ナレッジインデックスは対象プロジェクトのルートディレクトリ直下の `.lightrag/` に配置される。
AI Agent は「接続後すぐにプロジェクト固有のナレッジ検索が使える」状態を得る。
オペレーターは MCP サーバーの起動・停止のみを管理すればよく、ナレッジエンジンの独立した起動・停止操作は不要になる。

## 管理対象と状態遷移

### ナレッジエンジン（LightRAG）

- 状態: `停止中` → `起動処理中` → `稼働中` / `起動失敗`
- 遷移トリガー:
  - 停止中 → 起動処理中: MCP サーバーの起動処理が始まったとき
  - 起動処理中 → 稼働中: ナレッジエンジンが 30 秒以内に応答可能になったとき
  - 起動処理中 → 起動失敗: プロセス即時失敗、または 30 秒以内に応答可能にならなかったとき
  - 稼働中 → 停止中: MCP サーバーが正常停止されたとき
  - ※ MCP サーバーの異常終了（クラッシュ）時のプロセス残留管理は Phase 2 スコープ外
- 各状態の制約:
  - 起動処理中・起動失敗: lightrag_query に `LIGHTRAG_UNAVAILABLE` を返す
  - 停止中（MCP サーバー停止後）: lightrag_query に接続エラーを返す

### ナレッジインデックス（`.lightrag/`）

- 状態: `未構築` → `構築済み`（再インデックスで更新）
- 遷移トリガー:
  - 未構築 → 構築済み: インデックスコマンドが完了したとき
  - 構築済み → 構築済み（更新）: 再インデックスコマンドが完了したとき
- 各状態の制約:
  - 未構築: lightrag_query に `LIGHTRAG_INDEX_NOT_FOUND` エラーを返す
  - 更新は必ず同一 `.lightrag/` ディレクトリで行う

## ビジネスルール

| ID | ルール | 関連 AC |
|----|-------|--------|
| BR-EMBED-01 | ナレッジエンジンは MCP サーバーの起動・停止に連動する。オペレーターによる独立した手動起動は不要 | AC-F14-01, AC-F14-04a |
| BR-EMBED-02 | ナレッジインデックスは対象プロジェクトルート直下の `.lightrag/` に保存する | AC-F15-03, AC-F15-04 |
| BR-EMBED-03 | 起動待機時間は 30 秒固定。30 秒以内に稼働状態にならない場合、エラーコードと対処手順を返す（Phase 2 以降で環境変数化を検討） | AC-F14-02, AC-F14-03a, AC-F14-03b |
| BR-EMBED-04 | MCP サーバーの正常停止時、ナレッジエンジンプロセスを自動終了させる | AC-F14-04a, AC-F14-04b |
| BR-EMBED-05 | 再インデックス実行時、既存 `.lightrag/` を同一ディレクトリで更新する（新規ディレクトリの作成は禁止） | AC-F15-04 |
| BR-EMBED-06 | 異なる対象プロジェクトはそれぞれ独立した `.lightrag/` を持つ（プロジェクト間のインデックス混在は禁止） | AC-F15-01 |
| BR-EMBED-07 | コード検索ツールの戻り値のファイルパスは対象プロジェクトのディレクトリ配下のみとする | AC-F15-05 |
| BR-EMBED-08 | コード検索ツールは対象プロジェクトを分析の基準ディレクトリとして使用する（コードグラフインデックスの配置は Epic #4 が規定） | AC-F15-05 |

## 例外・禁止事項

| ID | 条件 | システムの振る舞い |
|----|------|----------------|
| EX-EMBED-01a | ナレッジエンジンがプロセス起動時に即時失敗した（ポート競合等） | 3 秒以内に stderr へエラーコードと対処手順を出力し、MCP レスポンスに構造化エラーを返す |
| EX-EMBED-01b | 30 秒以内にナレッジエンジンが応答可能にならなかった（起動タイムアウト） | タイムアウト確定後、即時に stderr へエラーコードと対処手順を出力し、MCP レスポンスに構造化エラーを返す |
| EX-EMBED-02 | ナレッジインデックスが未構築の状態でナレッジ検索が呼び出された | `LIGHTRAG_INDEX_NOT_FOUND` エラーコードとインデックス構築手順を含むエラーレスポンスを返す |
| EX-EMBED-03 | MCP サーバー停止後にナレッジ検索ツールが呼び出された | 接続エラーを返す |

## 業務イベント

| イベント名 | 発生条件 | 後続アクション |
|----------|---------|-------------|
| ナレッジエンジン自動起動完了 | MCP サーバー起動後、30 秒以内にナレッジエンジンが稼働状態になったとき | ナレッジ検索ツールが応答可能になる |
| ナレッジエンジン起動失敗 | プロセス即時失敗または 30 秒タイムアウトが確定したとき | エラーコードと対処手順が MCP レスポンスと stderr に出力される |
| ナレッジエンジン自動停止 | MCP サーバーが正常停止されたとき | ナレッジエンジンプロセスが終了する。未応答クエリがあった場合は接続エラーを返す |
| ナレッジインデックス作成完了 | インデックスコマンドが完了したとき | 対象プロジェクトルート直下の `.lightrag/` にインデックスファイル群が保存される |
| ナレッジインデックス更新完了 | 再インデックスコマンドが完了したとき | 既存の `.lightrag/` が更新される |
