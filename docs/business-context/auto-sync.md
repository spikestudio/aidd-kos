# Business Context: Auto Sync（コミット後の自動インデックス更新）

Epic: #26
対象 Phase: Operational Excellence（#2）
作成日: 2026-06-08

---

## 業務目的

オペレーターが git commit を行うたびに、手動操作なしにナレッジインデックスが自動更新される状態にする。
設定は一度だけ行えばよく、以降のコミットに対して継続的に機能する。
AI Agent は常に最新のナレッジグラフを参照して回答精度を維持できる。

---

## 登場人物

| 役割 | 説明 |
|------|------|
| AI 駆動開発エンジニア（オペレーター）| aidd-kos をプロジェクトに導入し、フック設定を一度だけ実施する人間の開発者 |
| AI Agent（Claude Code 等）| 常に最新のナレッジグラフを参照してクエリに回答する受益者 |

---

## ドメイン概念

| 用語 | 定義 | 備考 |
|------|------|------|
| lefthook | git フックをプロジェクトで共有管理するためのフック管理ツール | `lefthook.yml` に設定を記述し `lefthook install` で有効化する |
| post-commit フック | git commit 操作が完了した後に自動実行されるフック | コミット自体は既に確定しており、フックの成否がコミットに影響しない |
| フォールバック（\|\| true）| フック実行が失敗したとき、コミット操作に影響を与えずに処理を終了させるシェル制御 | `aidd-kos index \|\| true` の形式で使用する |
| 自動インデックス更新 | コミット後に手動操作なしで実行されるインデックス更新処理 | lefthook post-commit フック経由で `aidd-kos index`（差分インデックス）を実行する |

## ビジネスルール

| ID | ルール | 関連 AC |
|----|--------|---------|
| BR-ASYNC-01 | インデックス更新の失敗はコミット操作を中断させてはならない（`\|\| true` を必須とする）| AC-F[TBD]-03 |
| BR-ASYNC-02 | フックは post-commit（コミット完了後）に限定する。pre-commit（コミット前）はコミット操作をブロックするリスクがあるため禁止 | AC-F[TBD]-01 |
| BR-ASYNC-03 | ドキュメントに記載する設定例は `lefthook run post-commit` で動作確認できる形とする | AC-F[TBD]-02 |

---

## 例外・禁止事項

| ID | 条件 | 振る舞い |
|----|------|---------|
| EX-ASYNC-01 | `aidd-kos index` がエラーを返したとき（LightRAG 未起動・ネットワーク障害等）| `\|\| true` によりコミットに影響を与えず終了する。エラーはターミナルに表示されるがコミットは成立済み |
