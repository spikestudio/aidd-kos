# Business Context: Index Sync（インデックス差分更新）

Epic: #25
対象 Phase: Operational Excellence（#2）
作成日: 2026-06-04

---

## 業務目的

`aidd-kos index` が変更・追加・削除ファイルのみを対象に処理することで、
AI Agent が参照するナレッジインデックスを最小コストで最新状態に保つ。
従来の全件処理から差分処理への移行により、日常的なインデックス更新が現実的な操作になる。

---

## 登場人物

| 役割 | 説明 |
|------|------|
| AI 駆動開発エンジニア（オペレーター）| `aidd-kos index` を実行してインデックスを更新する人間の開発者 |
| AI Agent（Claude Code 等）| 更新されたインデックスを通じて最新のプロジェクト知識を検索・参照する受益者 |

---

## ドメイン概念

| 用語 | 定義 | 備考 |
|------|------|------|
| 差分インデックス | 前回インデックス処理時から変更・追加・削除されたファイルのみを対象とする更新操作 | `aidd-kos index`（デフォルト）で実行される |
| 全件再構築 | 変更の有無に関わらず全対象ファイルを強制再処理する更新操作 | `aidd-kos index --full` で実行される |
| インデックス状態ファイル | 前回インデックス処理時の対象ファイル一覧と処理日時を記録したローカルファイル。差分検出の基準として参照される | 対象プロジェクトの `.lightrag/` と同じくプロジェクト内に配置（.gitignore 済み）|
| 変更検出 | 前回インデックス処理時から変更・追加されたファイルを特定する操作 | — |
| 削除検出 | 前回インデックス処理時以降に削除されたファイルを特定し、インデックスから除去対象として識別する操作 | Q-SYNC-01 参照 |
| スキップ | 差分インデックス実行時に変更なしと判定され処理対象外となったファイル | API 呼び出しは発生しない |

---

## インデックス状態遷移

| 遷移 | トリガー |
|------|---------|
| 未初期化 → 最新 | `aidd-kos index` 初回実行（状態ファイルなし → 全件処理完了・状態ファイル作成）|
| 最新 → 差分あり | ファイルの変更・追加・削除が発生する |
| 差分あり → 最新 | `aidd-kos index` 実行（変更・追加ファイルの再処理完了かつ状態ファイル更新済み）|
| 未初期化 → 最新（--full）| `aidd-kos index --full` 実行（全件処理完了・状態ファイル作成）|
| 差分あり → 最新（--full）| `aidd-kos index --full` 実行（変更有無に関わらず全件処理完了・状態ファイル更新済み）|

## ビジネスルール

| ID | ルール | 関連 AC |
|----|--------|---------|
| BR-SYNC-01 | 状態ファイルが存在しない場合（初回実行）は全対象ファイルを処理する | AC-F[F01]-04 |
| BR-SYNC-02 | `--full` 指定時は状態ファイルの有無に関わらず全対象ファイルを処理する | AC-F[F03]-01・02 |
| BR-SYNC-03 | 変更がゼロ件の場合、API 呼び出しなしで正常終了する（エラーにしない）| AC-F[F01]-01 |
