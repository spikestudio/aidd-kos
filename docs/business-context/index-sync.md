# Business Context: Index Sync（インデックス差分更新）

Epic: #25
対象 Phase: Operational Excellence（#2）
作成日: 2026-06-04
更新日: 2026-06-05（設計方針変更: ローカル状態ファイル → LightRAG API を正規ソースとして採用）

---

## 業務目的

`aidd-kos index` が変更・追加・削除ファイルのみを対象に処理することで、
AI Agent が参照するナレッジインデックスを最小コストで最新状態に保つ。
従来の全件処理から差分処理への移行により、日常的なインデックス更新が現実的な操作になる。

---

## ドメイン概念

| 用語 | 定義 | 備考 |
|------|------|------|
| 差分インデックス | 前回インデックス処理時から変更・追加・削除されたファイルのみを対象とする更新操作 | `aidd-kos index`（デフォルト）で実行される |
| 全件再構築 | 変更の有無に関わらず全対象ファイルを強制再処理する更新操作 | `aidd-kos index --full` で実行される |
| LightRAG インデックス状態 | LightRAG の `/documents/paginated` API が返す、既にインデックス済みのドキュメント一覧（`file_path`・`id`・`updated_at` を含む）。差分検出の正規ソース。ローカルファイルは不要 | `aidd_kos/index.py` の `_fetch_indexed_docs()` が取得する |
| 変更検出 | LightRAG インデックス状態の `updated_at` とファイルシステムの `mtime` を比較して変更・追加ファイルを特定する操作 | UTC naive ISO8601 を UTC として扱う |
| 削除検出 | LightRAG インデックス状態にあるがファイルシステムに存在しないファイルを特定する操作 | `_detect_deleted()` が実装 |
| スキップ | 差分インデックス実行時に `mtime ≤ updated_at` と判定されAPI呼び出しが省略されたファイル | — |

---

## インデックス状態遷移

| 遷移 | トリガー |
|------|---------|
| 未インデックス → 最新 | `aidd-kos index` 初回実行（LightRAG が空 → 全件処理完了）|
| 最新 → 差分あり | ファイルの変更・追加・削除が発生する |
| 差分あり → 最新 | `aidd-kos index` 実行（変更・追加ファイルの再処理 + 削除ファイルの除去が完了）|
| 任意 → 最新（--full）| `aidd-kos index --full` 実行（変更有無に関わらず全件処理完了）|

## ビジネスルール

| ID | ルール | 関連 AC |
|----|--------|---------|
| BR-SYNC-01 | LightRAG インデックスが空の場合（初回実行）は全対象ファイルを処理する | AC-F29-04 |
| BR-SYNC-02 | `--full` 指定時は LightRAG インデックス状態を参照せず全対象ファイルを処理する | AC-F31-01・02 |
| BR-SYNC-03 | 変更がゼロ件の場合、API 呼び出しなしで正常終了する（エラーにしない）| AC-F29-01 |
| BR-SYNC-04 | LightRAG `/documents/paginated` のレスポンスが解析できない場合、全件処理にフォールバックし stderr に旨を通知する | — |
