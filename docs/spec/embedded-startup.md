## Feature: ナレッジ検索が MCP サーバー起動直後から利用可能になる (F01)

### Story S-01: MCP サーバーへの接続後すぐにナレッジ検索を使えること

As an AI Agent, I want to use ナレッジ検索を MCP サーバーへの接続後すぐに,
so that 別途ナレッジエンジンを手動で起動する手間なくプロジェクト知識の検索を開始できる.

根拠: [Problem Statement]

**AC:**

| ID | Given | When | Then | Note |
|----|-------|------|------|------|
| AC-F14-01 | MCP サーバーが起動している（ナレッジエンジンは手動起動していない） | ナレッジ検索ツールを呼び出したとき | `LIGHTRAG_UNAVAILABLE` エラーを返さずに検索結果が返されること | `skip:implicit-coverage` — `test_ac_f41_01_lightrag_query_available_immediately_after_startup` が実質的に同条件をカバー（AC-F41-01 の spec 定義は dual-process だが当該テストは single-process で embedded startup を検証）|
| AC-F14-02 | MCP サーバーが起動処理中（起動から 30 秒以内） | ナレッジ検索ツールを最初に呼び出したとき | MCP サーバー起動から 30 秒以内に検索結果が返されること（タイムアウトエラーなし） | `skip:timing-dependent` — 30秒タイムアウト計測は CI で不安定なため手動確認 |

### Story S-02: ナレッジエンジンの起動に失敗したとき対処方法付きのエラーを受け取ること

As an AI Agent, I want to receive 対処方法付きのエラー when ナレッジエンジンの起動に失敗したとき,
so that 問題の原因と解決手順を正確に把握でき、オペレーターへの適切な報告と対処が可能になる.

根拠: [AI 補完: エラーケース]

**AC:**

| ID | Given | When | Then |
|----|-------|------|------|
| AC-F14-03a | ナレッジエンジンがプロセス起動時に即時失敗した状態（ポート競合等） | MCP サーバーが起動処理を試みたとき | エラーコードと対処手順を含む構造化エラーレスポンスが返され、3 秒以内に stderr へ出力されること |
| AC-F14-03b | ナレッジエンジンが起動処理中（30 秒タイムアウト到達） | タイムアウトが確定したとき | タイムアウト確定後、即時に stderr へエラーコードと対処手順が出力され、MCP レスポンスに構造化エラーが返されること |

## Feature: 全ナレッジエンジンが対象プロジェクトを正しく参照する (F02)

### Story S-04: ナレッジ検索が現在作業中の対象プロジェクトの知識を返すこと

As an AI Agent, I want to receive ナレッジ検索が現在作業中の対象プロジェクトの知識を,
so that 他のプロジェクトの情報と混在せず当該プロジェクトに関連する正確な知識を取得できる.

根拠: [Problem Statement]

**AC:**

| ID | Given | When | Then |
|----|-------|------|------|
| AC-F15-01 | 対象プロジェクト A のみのインデックスを持つ MCP サーバーが起動している | ナレッジ検索ツールを呼び出したとき | 戻り値のすべてのエントリが対象プロジェクト A に由来すること（プロジェクト B のドキュメント識別子が含まれないこと） |
| AC-F15-02 | 対象プロジェクトにナレッジインデックスが未構築の状態（AI 補完: エラーケース） | ナレッジ検索ツールを呼び出したとき | エラーコード `LIGHTRAG_INDEX_NOT_FOUND` とインデックス構築手順を含むエラーレスポンスが返されること |

### Story S-05: ナレッジインデックスが対象プロジェクトのディレクトリに保存されていることを確認できること

As an AI 駆動開発エンジニア（オペレーター）, I want to confirm ナレッジインデックスが対象プロジェクトのディレクトリに保存されていることを,
so that プロジェクトごとにインデックスを独立して管理・バックアップできる.

根拠: [ユーザー入力]

**AC:**

| ID | Given | When | Then |
|----|-------|------|------|
| AC-F15-03 | 対象プロジェクトのルートディレクトリで aidd-kos index を実行したとき | ナレッジインデックスが作成されたとき | 対象プロジェクトの `.md`/`.txt` ファイルをテキスト API に送信し `file_sources` が対象プロジェクトルートからの相対パスであること |
| AC-F15-04 | 対象プロジェクトの `.lightrag/` にインデックスが存在する状態 | 再インデックスコマンドを実行したとき | 同じ対象プロジェクトのファイルが同じ相対パスで再送信されること（別プロジェクトのパスが混入しないこと）|
