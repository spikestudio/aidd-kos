# レビュー観点ファイル仕様

このディレクトリは `/aidd-review` が各ゲート（setup-complete/spec-approved/epic-complete/phase-complete）でマルチエージェントレビューを実施する際に参照する観点定義集である。

## ファイル構成

| ファイル | 対応ゲート | 観点数 |
|---------|----------|------|
| `setup-complete.md` | setup-complete | 6 |
| `spec-approved.md` | spec-approved | 9 |
| `epic-complete.md` | epic-complete | 12 |
| `phase-complete.md` | phase-complete | 3 |

合計 **30 観点**。

## ファイル構造

各ゲートファイルは「**1 ゲート = 1 レビュー = 1 ファイル**」の原則で構成される。
各ファイル内に `## 観点 N: ...` セクションが複数並ぶ。各観点は単独で 1 並列エージェントに渡される。

## 観点セクションのスキーマ

各観点は以下の構造で記述する:

```markdown
## 観点 N: [観点名] — [担当 subagent]

### レビュー対象
- [対象成果物・ファイルパス・Issue 等]

### チェック内容

1. **[項目1の見出し]**
   - [チェック詳細]

2. **[項目2の見出し]**
   - [チェック詳細]

...

### 重要度判定基準

- **Critical**: [この観点での Critical 判定条件]
- **Suggestion**: [この観点での Suggestion 判定条件]
- **OK Point**: [良い設計判断と判定する条件]

### 出力フォーマット

```
## 観点 N レビュー結果: [観点名]

### Critical
- [問題の説明]
  - 該当箇所: [ファイル・行]
  - 理由: [なぜ問題か]
  - 修正方針: [具体的な対策]

### Suggestions
- ...

### OK Points
- ...

### Analysis Notes
- レビュー限界: ...
- 不確実性: ...
```
```

## `/aidd-review` での読み込み手順

1. ゲートを確定（setup-complete/spec-approved/epic-complete/phase-complete のいずれか）
2. 対応するファイル全文を読み込む（`[ゲート名].md`）
3. ファイル内の `## 観点 N: ...` セクションを正規表現で分割
4. 各観点セクションをそれぞれ独立した `Agent(subagent_type=<担当>, prompt=<セクション内容>)` で並列起動
5. 全エージェント完了後、結果を Critical / Suggestions / OK Points で集約

## 観点を追加する際の手順

1. 該当ゲートのファイルを開く
2. 既存観点の末尾に `## 観点 N+1: ...` セクションを追加
3. 上記スキーマに従って記述
4. ファイル冒頭の観点数を更新
5. このファイル（README.md）の観点数も更新

## 重要なルール

- **観点間の重複を避ける** — 同じ subagent でも、ゲートに応じて見る対象を明確に区別する
- **観点は単独で完結する** — 他観点への依存・他ファイル前提の知識を含めない
- **placeholder 禁止** — 「TBD」「後で書く」「省略」等を含めない
- **対象成果物を必ず明記** — 抽象的な「コード全体」ではなく具体的なパス・Issue 番号を書く
