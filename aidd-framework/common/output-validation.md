# 出力検証ルール（共通骨格）

MANDATORY: 親スキルはサブスキル（Sub1/Sub2）の出力を受け取った直後、以下を検証する。

スキル固有の検証項目（Sub1 必須セクション・Sub2 ペルソナ名・固有 placeholder パターン・固有検証）は、各スキルの `SKILL.md` の「## 出力検証差分」セクションで定義する。

---

## 共通検証チェック

| チェック項目 | NG 時のアクション |
|------------|----------------|
| 必須セクションが揃っているか | サブスキル再実行（不足項目を feedback に追加） |
| placeholder が残っていないか（共通 + スキル固有パターン） | サブスキル再実行 |
| Markdown 構造が壊れていないか | サブスキル再実行 |
| Analysis Notes が空でないか | サブスキル再実行 |
| Sub2 の「Critical」「Suggestions」「OK Points」のいずれかが欄外表記なしで欠落 | サブスキル再実行 |

---

## Sub2 の共通必須セクション

スキル固有のサブ2 ペルソナによらず、以下は全スキル共通で必須:

- `Critical` セクション（0件なら「該当なし」明記）
- `Suggestions` セクション（0件なら「該当なし」明記）
- `OK Points` セクション（0件なら「該当なし」明記）
- `Analysis Notes` セクション（自己批判を含む）

---

## 共通 placeholder 判定基準

以下のパターンが出力に含まれる場合は placeholder NG とみなす:

```
TBD
TODO
[項目名]
[理由]
未記入
未定
(記述予定)
```

MANDATORY: スキル固有の placeholder パターンは `SKILL.md` 「## 出力検証差分」で追加定義する。

---

## 再実行ポリシー

- 最大 3 回まで再実行する
- 3 回失敗した場合は停止してユーザーに状況を報告し、手動介入を求める
- 再実行時は前回の出力と「ここが不足/不正」という feedback を必ず渡す
- NEVER: 検証 NG のまま黙って続行する
- NEVER: NG 項目をユーザーに隠す

---

## 検証失敗時の報告フォーマット

```
[出力検証NG] Sub{1|2} の出力に問題を検出しました。

検出項目:
- [NG チェック項目]

サブスキルを再実行します（{N}/3 回目）。
```

---

## スキル側で定義する差分

スキルは自身の `SKILL.md` 「## 出力検証差分」セクションで以下を定義する:

| スロット | 内容 | 必須 |
|---------|------|------|
| `SUB1_REQUIRED_SECTIONS` | Sub1 の必須セクション（スキル別テーブル or 共通記述） | ✅ |
| `SUB2_PERSONA` | Sub2 のペルソナ名（例: `aidd-architect`, `aidd-product-owner`） | ✅ |
| `EXTRA_PLACEHOLDERS` | スキル固有の placeholder パターン（例: `[集約名]`, `[テストファイル]`） | 任意 |
| `EXTRA_VALIDATIONS` | スキル固有の追加検証項目（例: TDD の Red/Green 確認、AC-ID 含有） | 任意 |
