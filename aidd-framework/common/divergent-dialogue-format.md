## 対話ステートと対応パターン

FRAMEWORK.md「探索的フェーズの対話設計原則」で定義した 4 状態（`Exploring` / `Confirming` / `Refining` / `Converging`）それぞれの対話パターンを定義する。

---

### Exploring 状態 — 仮説提示パターン

**目的**: ユーザーのふわっとした入力から問題・価値の仮説を形成・提示する。

**パターン:**

```
[受け取り]
ユーザーの入力（課題でも構想でも可）を受け取る。

[推論]
AI は以下を推論する:
- Problem Statement: 誰が何に困っているか
- Value Proposition: 誰に何を提供するか
入力が課題駆動の場合: Problem Statement を先に確定し Value Proposition を導出する。
入力がアイデア駆動の場合: Value Proposition を先に確定し Problem Statement を逆算する。

[提示]
推論した仮説を 1 つのみ提示する:
「〇〇という問題を解決しようとしていますか？（提案の根拠: ...）」
複数の解釈が可能な場合も、最も有力な 1 つを選んで提示する。選択肢を列挙しない。

[確認]
ユーザーに「この理解で合っていますか？」と2択で確認する。
```

**禁止事項:**
- NEVER: 「A 案、B 案、C 案のどれですか？」という複数選択肢の提示
- NEVER: 「課題はありますか？」という空の問いかけ（推論した仮説を先に提示する）

**質問規約（ADR-016 参照）:**
- `Exploring` 状態では `docs/questions/` への書き出し不要。対話内で直接質問してよい。

---

### Confirming 状態 — 指向性質問パターン

**目的**: 問題仮説を確定させ、調査・要件具体化の準備を行う。

**パターン:**

```
[不明瞭点の特定]
AI は問題仮説の確認に必要な情報のうち、まだ不明な点を1〜2件に絞って特定する。

[指向性質問]
「この点を確認したいので教えてください: [具体的な質問]」
質問は「YES/NO」や「A/B」ではなく、ユーザーが自由に答えられる形にする。
1 回の質問は 1 件に絞る。

[調査計画の提示（必要な場合）]
「この点を確認するために外部調査を行います: [調査目的・範囲]」
ユーザーの承認後にリサーチを実行する（承認前に開始しない）。
```

**禁止事項:**
- NEVER: 調査計画をユーザー承認前に実行する
- NEVER: 1 回に 3 件以上の質問をまとめて投げる

**質問規約（ADR-016 参照）:**
- `Confirming` 状態では `docs/questions/` への書き出し不要。対話内で直接質問してよい。

---

### Refining 状態 — 指摘具体化パターン

**目的**: ユーザーのふわっとした指摘を具体的な改善案に変換する。

**パターン:**

```
[指摘の受け取り]
「なんか違う」「もっと大きく」「なんか違和感がある」等のふわっとした指摘を受け取る。

[推論]
AI は指摘から変更すべき要素を推論する:
- 何が問題か（的外れな仮説？スコープが狭い？別のペルソナが対象？）
- 何を変えると解消されるか

[改善案の提示]
「〇〇を変えると△△になります」という具体的な改善案を 1 つ提示する。
「何が違うと感じますか？」と逆に質問して返さない（推論して提示が先）。

[確認]
「この方向で修正しますか？」と2択で確認する。
```

**禁止事項:**
- NEVER: ふわっとした指摘に対して「どこが違うと感じますか？」と質問で返す
- NEVER: 複数の修正案を列挙して選ばせる

---

### Converging 状態 — PRD 確認パターン

**目的**: PRD 全セクションが充足されていることを確認し、spec-approved 承認を得る。

**パターン:**

```
[充足確認]
PRD の全必須セクションを確認する:
- Problem Statement / Value Proposition
- ユーザー（Primary / Secondary / Negative）
- ユーザーストーリー（全件）
- ユーザージャーニー
- 成功指標（数値・期限）
- Won't Have（1件以上）

[不足セクションへの対応]
不足があれば「〇〇セクションが未確定です。AI が推論した案: [...]。これで進めますか？」

[最終承認要求]
「PRD は承認できる状態です。spec-approved を承認しますか？」
```

**質問規約の復活（ADR-016 参照）:**
- `Converging` 状態では `docs/questions/` 書き出し規約を適用する（非同期確認が適切な段階）。

## SKILL.md での参照方法

探索的フェーズスキルの Step 開始前に本ファイルを参照する:

```markdown
## references/ の参照

| reference | 用途 |
|-----------|------|
| `../../aidd-framework/common/divergent-dialogue-format.md` | 探索的フェーズの対話パターン（仮説提示・指向性質問・指摘具体化・PRD確認）|
```

**参照タイミング**: 探索的フェーズの各 Step を開始する前に Read して対話パターンを確認する。

## presentation-format.md との使い分けまとめ

| 観点 | `divergent-dialogue-format.md`（本ファイル）| `presentation-format.md` |
|------|------------------------------------------|--------------------------|
| 適用フェーズ | 探索的（What to build）| 収束的（How to build）|
| AI の役割 | ガイド（仮説提示・推論・合意形成）| 実行者（成果物生成・提示・承認取得）|
| 出力の性質 | 叩き台（修正前提）| 確定成果物（承認後は変更しない）|
| 選択肢の提示 | 禁止（1 つのみ提示）| 2択のみ（承認 / 修正）|
| 質問規約 | Exploring/Confirming: 直接質問可（ADR-016）| docs/questions/ に書き出す |
